# current_status

## 現在動作している機能

- React + Vite アプリとして起動・ビルドできます。
- 初回起動時、モンスター名・作物名・ほ場名・作物カテゴリを入力してタマゴを作成できます。
- 作成したモンスター、エサ、エサやり記録、現在画面は `localStorage` に保存されます。
- ホーム画面で現在のモンスター、作物バッジ、ステータス、メッセージを表示できます。
- Canvas ベースのピクセルモンスター描画が動作しています。
- 水やり、施肥、観察、収穫、防除、その他のクイックエサやりが動作しています。
- エサやり時に段階的な食事アニメーションとリアクションアニメーションが表示されます。
- エサカテゴリごとのルールに基づき、元気・機嫌・健康・成長度が更新されます。
- 成長度に応じて 0〜7 の成長段階が判定され、進化時にはメッセージが変わります。
- エサ作成画面で、カテゴリ・名前・メモを入力してエサ一覧へ保存できます。
- 記録画面で、クイックエサやりによって生成された履歴を確認できます。

## 未実装機能

- エサ一覧で選択した作成済みエサを、実際にモンスターへ与える処理は未接続です。現在はホーム画面へ戻るだけです。
- 作成済みエサを使ったエサやり履歴の作成は未実装です。
- `LocalBrain` は名前に反して、現時点ではルールベースの反応判定です。
- モンスター個体差、作物カテゴリ、季節、ほ場状態、外部農業データによる反応補正は未実装です。
- サンプルエサデータは定義されていますが、初期状態への自動投入には使われていません。
- 作成済みエサ、モンスター、記録の編集・削除機能は未実装です。
- アプリ状態のリセットUIは未実装です。

## 現在の課題

- `App.tsx` の `handleFeedFromList` が給餌ロジックへ接続されておらず、作成したエサを一覧から選んでもステータス更新や記録作成が行われません。
- 給餌シーケンスは `HomeScreen` 内のクイックエサやりに閉じているため、作成済みエサを再利用するには処理の受け渡し設計が必要です。
- `src/data/initialState.ts` と `src/app/appState.ts` に初期状態定義が分かれており、どちらを正とするか整理が必要です。
- localStorage のデータ構造にバージョン移行処理がないため、モデル変更時に古い保存データとの互換性問題が出る可能性があります。
- TypeScript のビルド成果物や Vite の一時ファイルがリポジトリに混入していたため、Git 管理対象から外して `.gitignore` に追加しました。
